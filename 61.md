# Socket编程
## 处理部分读写操作
1. 问题：部分读取与部分写入
- 部分读取：`read()`返回的字节数少于请求的字节数
原因：信号中断、数据尚未全部到达、字节流边界等
- 部分写入：`write()`返回的字节数少于请求的字节数
原因：信号中断、缓冲区满、非阻塞模式等
2. 解决方案：`readn()`和`writen()`
```c
ssize_t readn(int fd, void *buf, size_t n);
ssize_t writen(int fd, const void *buf, size_t n);
```
- 功能：确保完整的读取和写入制指定字节数的数据
- 实现原理：循环调用`read()/write()`，直到传输完所有字节或遇到错误
- 返回值：成功返回`n`，失败返回 -1，文件结束返回实际读取的字节数（小于`n`）
## `recv()`和`send()`：带标志的 I/O
与`read()/write()`相比，`recv()/send()`多了一个`flags`参数
```c
ssize_t recv(int sockfd, void *buf, size_t len, int flags);
ssize_t send(int sockfd, const void *buf, size_t len, int flags);
```
常用标志：

| 标志             | 作用                        |
| -------------- | ------------------------- |
| `MSG_DONTWAIT` | 非阻塞 I/O（单次调用）             |
| `MSG_OOB`      | 发送/接收带外数据（紧急数据）           |
| `MSG_PEEK`     | 窥探数据（读取但不从缓冲区移除）          |
| `MSG_WAITALL`  | 等待完整请求长度的数据（类似 `readn()`） |
## `sendfile()`:高效的文件传输
```c
ssize_t sendfile(int out_fd, int in_fd, off_t *offset, size_t count);
```
1. 功能：将文件内容直接拷贝到套接字
2. 高效原理：
- 传统方式：文件 → 用户空间缓冲区 → 套接字（两次数据拷贝）
- `sendfile()`：文件 → 套接字（内核空间直接拷贝，无用户空间参与）
- 适用场景：Web 服务器发送静态文件、FTP 服务器
## 连接管理高级函数
1. `shutdown()`：精细控制连接关闭
```c
int shutdown(int sockfd, int how);
```

| `how` 值     | 作用                                  |
| ----------- | ----------------------------------- |
| `SHUT_RD`   | 关闭读端（后续 `read()` 返回 EOF）            |
| `SHUT_WR`   | 关闭写端（后续 `write()` 产生 SIGPIPE/EPIPE） |
| `SHUT_RDWR` | 关闭读写两端（类似 `close()`）                |
与`close()`：的区别
- `close()`：只有当所有文件描述符引用计数为 0 时才关闭连接
- `shutdown()`：**立即关闭指定的方向**，无论是否有其他描述符指向该套接字
2.  `getsockname()` 和 `getpeername()`
```c
int getsockname(int sockfd, struct sockaddr *addr, socklen_t *addrlen);
int getpeername(int sockfd, struct sockaddr *addr, socklen_t *addrlen);
```
- `getsockname()`：获取套接字绑定的**本地地址**
- `getpeername()`：获取连接的**对端地址**

2. `TIME_WAIT`状态
- 持续时间：2 × MSL
- 作用：确保最后一个ACK被对端接收，实现可靠关闭；防止旧连接中的延迟报文被新连接错误接收
- 问题：服务器主动关闭连接后进入 `TIME_WAIT`，如果立即重启并尝试绑定同一端口，会出现 "Address already in use" 错误
3. `SO_REUSEADDR` 套接字选项
```c
int optval = 1;
setsockopt(sockfd, SOL_SOCKET, SO_REUSEADDR, &optval, sizeof(optval));
```
- 作用：允许重用于处于`TIME_WAIT`状态的端口
- 用法：服务器在`bind()`之前设置此选项
## 套接字选项：`getsockopt()` / `setsockopt()`
```c
int getsockopt(int sockfd, int level, int optname, void *optval, socklen_t *optlen);
int setsockopt(int sockfd, int level, int optname, const void *optval, socklen_t optlen);
```
常用选项：

| 级别            | 选项                        | 作用                               |
| ------------- | ------------------------- | -------------------------------- |
| `SOL_SOCKET`  | `SO_REUSEADDR`            | 重用 TIME_WAIT 端口                  |
| `SOL_SOCKET`  | `SO_KEEPALIVE`            | 空闲时发送心跳包检测连接                     |
| `SOL_SOCKET`  | `SO_RCVBUF` / `SO_SNDBUF` | 设置接收/发送缓冲区大小                     |
| `SOL_SOCKET`  | `SO_LINGER`               | 控制 `close()` 的行为（立即关闭 vs 等待数据发送） |
| `IPPROTO_TCP` | `TCP_NODELAY`             | 禁用 Nagle 算法（减少延迟）                |
## `accept()`返回套接字的继承规则
在Linux上，`accept()`返回的新套接字不会继承：
- 监听套接字的信号驱动 I/O 相关设置
- 打开文件状态标记
- 文件描述符标记
但会继承：
- 已设置的套接字选项
## UDP的使用场景

| 场景        | 原因                       |
| --------- | ------------------------ |
| 实时多媒体     | 丢包可容忍，延迟更重要；TCP 的重传会导致卡顿 |
| DNS查询     | 单次请求-响应，不需要建立连接开销        |
| 广播/多播     | UDP 原生支持，TCP 不支持         |
| 简单请求-响应协议 | 如 NTP、DHCP、SNMP          |
| 游戏状态同步    | 状态信息有冗余，丢包可接受            |
