# SOCKET：TCP/IP 网络基础

## TCP/IP分层模型

TCP/IP是一个分层的联网协议栈，每一层负责不同的功能

- 应用层（HTTP、SSH）：用户应用程序协议
- 传输层（TCP、UDP）：端到端通信、提供进程间通信（端口）
- 网络层（IPv4、IPv6）：主机间路由和寻址、无连接不可靠
- 链路层（以太网、Wi-Fi）：物理介质上的帧传输

```bash
应用数据
    ↓
[传输层头 + 数据] → 段（Segment）/数据报（Datagram）
    ↓
[网络层头 + 传输层段] → 包（Packet）
    ↓
[链路层头 + 网络层包] → 帧（Frame）
    ↓
物理介质传输
```

## IP层（网络层）

1.IP的基本特性

- 作用：在源主机和目的主机之间传输数据报，负责路由和寻址
- 核心特性：无连接、不可靠、尽力而为

2.IPv4 vs IPv6
地址长度：IPv4(32位)   IPv6(128位)

## 传输层协议

在IP层之上，存在多种传输层协议，其中使用最多的是UDP和TCP

1.UDP

- 类型：数据报协议
- 可靠性：不可靠（可能丢包、乱序、重复）
- 连接性：无连接
- 消息边界：保留
- 开销：低

2.TCP

- 类型：可靠的、面向连接的字节流协议
- 可靠性：保证数据按序、无重复、无丢失的到达
- 连接性：面向连接
- 消息边界：无
- 开销：较高

## TCP的核心机制

1.数据打包与分段

- 接收端按正确顺序组装分段
- 确认ACK已接收的数据
- 检测重复和丢失的分段

2.确认机制

- 接收端发送ACK确认已收到的数据
- 发送端位收到ACK时会重传数据

3.流量控制

- 目的：防止快速的发送者压垮慢速的接收者
- 机制：使用滑动窗口
- 接收端通过`rwnd`（接收窗口）通告自己还能接受多少数据
- 发送端根据窗口大小控制发送速率

4.拥塞控制

- 目的：防止快速的发送者压垮整个网络
- 慢启动：开始时指数增长发送速率
- 拥塞避免：达到阈值后线性增长
- 快速重传：检测到丢包时立即重传
- 快速恢复：避免进入慢启动
