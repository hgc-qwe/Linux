# Internet Domain Socket 编程

## Internet Domain Socket 地址
1. 地址组成
- IP地址：标识网络中的主机
IPV4 ：32位数字
IPv6：128位数字
- 端口号：标识主机上的特定应用程序（16位）
2. IPv4地址结构（`sockaddr_in`)
```c
struct sockaddr_in {
    sa_family_t    sin_family;   // AF_INET
    in_port_t      sin_port;     // 端口号（网络字节序）
    struct in_addr sin_addr;     // IPv4 地址
};

struct in_addr {
    uint32_t       s_addr;       // IPv4 地址（网络字节序）
};
```
3. IPv6地址结构
```c
struct sockaddr_in6 {
    sa_family_t    sin6_family;   // AF_INET6
    in_port_t      sin6_port;     // 端口号（网络字节序）
    uint32_t       sin6_flowinfo; // 流信息
    struct in6_addr sin6_addr;    // IPv6 地址
    uint32_t       sin6_scope_id; // 范围 ID
};

struct in6_addr {
    uint8_t        s6_addr[16];   // IPv6 地址（16 字节）
};
```
## 字节序与跨平台问题
1. 问题的根源
不同的计算机架构使用不同的方式表示数据类型
- 大端序：高位字节存储在低地址（网络字节序）
- 小端序：低位字节存储在低地址（x86架构）
- 不同架构对`int`、`long`等类型可能使用不同的字节数`
2. 网络字节序
- 所有在网络中传输的二进制数据必须使用网络字节序
- 转换函数
```c
#include <arpa/inet.h>

uint32_t htonl(uint32_t hostlong);   // 主机 → 网络（32 位）
uint16_t htons(uint16_t hostshort);  // 主机 → 网络（16 位）
uint32_t ntohl(uint32_t netlong);    // 网络 → 主机（32 位）
uint16_t ntohs(uint16_t netshort);   // 网络 → 主机（16 位）
```
3. IP地址转换函数
```c
// 字符串 ↔ 二进制地址
int inet_pton(int af, const char *src, void *dst);        // 文本 → 二进制
const char *inet_ntop(int af, const void *src, char *dst, socklen_t size);  // 二进制 → 文本
```
## 数据传输与数据表示问题
1. 异构系统间
当在不同架构的及其间传输数据时，需要解决：
- 字节序差异
- 数据类型大小差异
- 结构体对齐差异
- 浮点数表示差异
2. 信号编集
信号编集是将数据转换为独立于架构的表示形式的过程
- 常见的解决方案：

| 方案               | 描述           |
| ---------------- | ------------ |
| 文本编码             | 将所有数据编码成文本形式 |
| XDR              | 标准化二进制编码     |
| ASN.1            | 国际标准         |
| Protocol Buffers | 序列化协议        |
| JSON/XML         | 文本格式         |
3. 文本编码
- 将所有传输的数据编码成文本形式
- 字段之间时用预定义字符分割
## 名称解析
1. 为什么使用名称而不是数字
- 名字更容易记忆
- 名字更稳定
- 支持负载均衡
2. 主机名解析函数
```c
#include <sys/types.h>
#include <sys/socket.h>
#include <netdb.h>

int getaddrinfo(const char *node,           // 主机名或 IP 地址字符串
                const char *service,        // 服务名或端口号字符串
                const struct addrinfo *hints, // 筛选条件
                struct addrinfo **res);     // 返回结果链表

void freeaddrinfo(struct addrinfo *res);    // 释放结果
const char *gai_strerror(int errcode);      // 错误信息
```
`struct addrinfo`结构：
```c
struct addrinfo {
    int              ai_flags;      // 标志位
    int              ai_family;     // AF_INET 或 AF_INET6
    int              ai_socktype;   // SOCK_STREAM 或 SOCK_DGRAM
    int              ai_protocol;   // 协议（通常 0）
    socklen_t        ai_addrlen;    // 地址长度
    struct sockaddr *ai_addr;       // 地址结构
    char            *ai_canonname;  // 规范主机名
    struct addrinfo *ai_next;       // 下一个结果
};
```
