# UNIX Domain Socket

## UNIX Domain Socket概述

1.基本概念

- 定义：UNIX Domain Socket是一种在同一主机上运行的进程间通信机制
- 支持类型：流Socket、数据报Socket

2.地址类型

```c
struct sockaddr_un {
    sa_family_t sun_family; // AF_UNIX
    char sun_path[108]; // 路径名（通常108字节， 包括结束字符）
};
```

3.文件类型
Socket 文件是特殊文件（文件类型为 s，在 ls -l 中显示为 srwxr-xr-x ）

## UNIX Domain Socket 编程模型

1.流Socket：

- 服务器端

```c
int sfd = socket(AF_UNIX, SOCK_STREAM, 0);
struct sockaddr_un addr;
addr.sun_family = AF_UNIX;
strcpy(addr.sun_path, "/tmp/mysocket");
bind(sfd, (struct sockaddr *)&addr, sizeof(addr));
listen(sfd, 5);
int cfd = accept(sfd, NULL, NULL);
// 使用 read()/write() 通信
close(cfd);
close(sfd);
unlink("/tmp/mysocket");  // 删除 Socket 文件
```

- 客户端

```c
int sfd = socket(AF_UNIX, SOCK_STREAM, 0);
struct sockaddr_un addr;
addr.sun_family = AF_UNIX;
strcpy(addr.sun_path, "/tmp/mysocket");
connect(sfd, (struct sockaddr *)&addr, sizeof(addr));
// 使用 read()/write() 通信
close(sfd);
```

2.数据报Socket

- 服务器端

```c
int sfd = socket(AF_UNIX, SOCK_DGRAM, 0);
struct sockaddr_un addr;
addr.sun_family = AF_UNIX;
strcpy(addr.sun_path, "/tmp/mysocket");
bind(sfd, (struct sockaddr *)&addr, sizeof(addr));
// 使用 recvfrom()/sendto() 通信
char buf[1024];
struct sockaddr_un client_addr;
socklen_t len = sizeof(client_addr);
recvfrom(sfd, buf, sizeof(buf), 0, 
         (struct sockaddr *)&client_addr, &len);
sendto(sfd, "response", 9, 0, 
       (struct sockaddr *)&client_addr, len);
close(sfd);
unlink("/tmp/mysocket");
```

- 客户端

```c
int sfd = socket(AF_UNIX, SOCK_DGRAM, 0);
struct sockaddr_un server_addr;
server_addr.sun_family = AF_UNIX;
strcpy(server_addr.sun_path, "/tmp/mysocket");
sendto(sfd, "hello", 6, 0, 
       (struct sockaddr *)&server_addr, sizeof(server_addr));
// 接收响应...
close(sfd);
```

- 服务端必须`bind()`到一个路径

## `socketpair()`：创建一对相互连接的Socket

1.基本概念

- 作用：创建一对相互连接的UNIX Domain Socket
- 用途：父子进程间通信（`fork()`）

2.函数

```c
int socketpair(int domain, int type, int protocol, int sv[2]);
```

## Linux特有的抽象Socket命名空间

1.基本概念

- 抽象Socket名：将UNIX Domain Socket绑定到不存在于文件系统中的名字
- 优势：自动清理、无需文件权限检查、无文件名冲突

2.特点

- `sun_path`中\0后所有字节（最多107字节）
- 当所有引用该Socket的文件描述符关闭时自动消失
