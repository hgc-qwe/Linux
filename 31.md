# 总结

## 线程安全的基本概念

1.定义：一个函数如果可以被多个线程同时调用而不会产生数据竞争或未定义行为，则该函数是线程安全的
2.非线程安全的常见原因：

- 使用全局变量或静态变量
- 返回指向静态缓冲区的指针
- 在多次调用之间维护状态

## 实现线程安全的两种基本方法

1.使用互斥量保护整个函数

- 优点：实现简单，适用于任何函数
- 缺点：兵法性能严重下降，同一时刻只有一个线程能执行该函数

2.仅保护临界区

- 优点：并发性能好，多个线程可以同时执行非临界区的代码
- 适用场景：函数的大部分操作是线程独立的，只有少量共享数据需要保护

## 可重入函数

1.定义：一个函数如果不使用任何共享数据，并且不调用非可重入函数，则称为可重入函数
2.特性：可重入函数天然是线程安全的

## 处理固有非线程安全函数的两种技术

1.线程特有数据

- 概念：为每个线程维护一份独立的、持久的数据副本。每个线程通过相同的键访问自己独有的数据副本
- 适用场景：需要在线程生命周期内持续存在的数据

```c
pthread_key_t key;

// 创建一个键，并指定解构函数
pthread_key_create(&key, destructor_function);

// 设置当前线程的特有数据
pthread_setspecific(key, value_ptr);

// 获取当前线程的特有数据
void *value = pthread_getspecific(key);

// 删除键
pthread_key_delete(key);
```

- 在函数第一次被调用时，为当前线程分配数据并关联到键，后续调用直接获取该数据
- 优点：不改变函数接口，每个线程有独立副本
- 缺点：实现相对复杂，有性能开销

2.线程局部存储

- 概念：使用编译器支持的关键字，将全局和静态变量声明为“每个线程独立”

```c
static __thread int cached_value = 0;
```

- 优点：简洁，性能好，无需手动管理键和解构函数
- 缺点：需要编译器支持
