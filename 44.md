# 总结

## 管道的基本概念

1.定义：UNIX系统上最早出现的IPC方法，由shell和许多应用程序广泛使用
2.本质：内核维护的一个单向数据通道，本质是一个内存缓冲区
3.创建：

```c
int pipe(int pipefd[2]);
```

成功返回0,失败返回-1

## 管道的核心特点

- 单向通信：数据只能从一个方向流动。需要双向通信需创建两个管道
- 亲缘进程：只能用于有亲缘关系的进程。通过`fork`继承文件描述符实现
- 字节流：无结构的字节流，没有消息边界。读取需自行解析数据格式
- 容量有限：管道有固定大小的缓冲区。写满是写入操作会阻塞
- 原子性：写入小于等于PIPE_BUF字节是原子的。避免数据交错的关键
- 自动同步：读写操作自然提供同步机制。可用作简单的同步工具

## PIPE_BUF与原子性

1.PIPE_BUF：写入管道时保证原子操作的最大字节数

- Linux上通常为4096字节

2.原子写入的重要性：

- 写入小于等于PIPE_BUF：内核保证写入操作是原子的，数据不会与其他进程的写入交错
- 写入大于PIPE_BUF：内核可能会拆分写入，与其他进程的写入交错

## `popen()`和`pclose()`—— 简化版管道

1.作用：允许程序向shell命令传输数据或从shell命令读取数据，无需手动处理创建管道、执行shell、关闭描述符的细节
2.函数

```c
FILE *popen(const char *command, const char *type);
int pclose(FILE *stream);
```

## FIFO的基本概念

1.定义：与管道类似，，但在文件系统中有一个名称，可以通过路径名访问
2.创建

```c
int mkfifo(const char *pathname, mode_t mode);
```

3.非阻塞打开

- 读：立即成功
- 写：如果没有读者：打开失败

## 文件描述符复制

1.目的：将管道的读写端重定向到标准输入/输出，实现过滤器程序
2.函数：

- `dup()`：复制到最小的可用描述符
- `dup2()`：复制到指定描述符
- `fcntl(fd, F_DUPFD, ...)`

## 处理消息边界

1.特殊分割符：使用特殊字符如换行符分隔消息
2.带长度前缀：每条消息前加上长度字段（先读长度。再读数据）
3.固定长度消息：每条消息大小相同，按固定字节数读取

## 客户端-服务器设计模式

- 众所周知的地址：服务器使用一个众所周知的FIFO路径名，供所有客户端连接
- 迭代式服务器：服务器一次只服务一个客户端，处理完后再服务下一个
- 并发式服务器：服务器为每个客户端创建新进程或线程处理
- 私有通信通道：客户端创建自己的私有FIFO，让服务器写回响应
