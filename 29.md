# 总结

1.线程的基本概念

- 定义：线程是进程内的一个执行流，是CPU调度的基本单位
- 线程的优势：
创建开销小、线程间通信简单、资源占用小、适合并行计算

2.进程与线程的资源对比

- 内存空间
进程（父子）：独立（写时复制或完全独立）
线程（同进程内）：完全共享

- 全局变量
进程（父子）：独立副本
线程（同进程内）：共享

- 文件描述符
进程（父子）：共享（fork后继承）
线程（同进程内）：共享

- 信号处理
进程（父子）：独立
线程（同进程内）：共享（但每个线程有独立信号掩码）

- 当前工作目录
进程（父子）：独立
线程（同进程内）：共享

- 用户/组ID
进程（父子）：独立
线程（同进程内）：共享

- 栈
进程（父子）：独立
线程（同进程内）：每个线程独立（但位于同一地址空间内）

- 程序计数器
进程（父子）：独立
线程（同进程内）：每个线程独立

3.基本线程管理

- 线程创建

```c
int pthread_create(pthread_t *thread, 
                   const pthread_attr_t *attr,
                   void *(*start_routine) (void *), 
                   void *arg);
```

- 线程终止

```c
void pthread_exit(void *retval);
```

- 等待线程终止

```c
int pthread_join(pthread_t thread, void **retval);
```

- 线程标识

```c
pthread_t pthread_self(void);   
int pthread_equal(pthread_t t1, pthread_t t2);  
```

4.线程属性`(pthread_attr_t)`

- 分离状态：设置为`PTHREAD_CREATE_DETACHED`后，线程终止时会自动清理资源
- 栈的大小和地址
- 调度策略和优先级

5.线程安全和可重入

- 线程安全：一个函数可以被多个线程同时调用而不产生错误
- 可重入：一个函数可以被多个执行流同时调用，且每次调用都产生正确的结果

6.线程与信号

- 每个线程都有独立的信号掩码
- 进程级别的信号会被发送到任意一个没有阻塞该信号的线程
- 在多线程程序中，最好使用专门的线程来处理所有信号，或避免使用信号
