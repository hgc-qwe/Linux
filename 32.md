# 总结

## 线程取消的基本概念

1.定义：线程取消是指一个线程向另一个线程发送请求，要求其终止运行的进制
2.函数

```c
int pthread_cancel(pthread_t thread);
```

- 向指定线程发送取消请求
- 立即返回，不等待目标线程实际终止
- 成功返回0,失败返回错误号

## 线程的取消状态

每个线程有两个与取消相关的属性：可取消状态和可取消类型
1.可取消状态

- 启用：默认状态。线程可以响应取消请求
- 禁用：线程忽略所有取消请求。请求会保持挂起状态，直到线程重新启用取消状态
- 函数：

```c
int pthread_setcancelstate(int state, int *oldstate);
```

2.可取消类型

- 延迟取消：线程仅在到达取消点时才响应取消请求
- 异步取消：线程随时可能响应取消请求
- 函数：

```c
int pthread_setcanceltype(int type, int *oldtype);
```

3.异步取消的危险

线程可能在持有互斥量、分配内存、更新数据结构的过程中被取消，导致：

- 互斥量永远无法解锁
- 内存泄漏
- 数据结构不一致

## 取消点

取消点是预定义的函数调用点，当线程到达这些点时，会检查是否有挂起的取消请求，如果有，则线程终止

1.常见的取消点函数：

- 线程相关：`pthread_join()`, `pthread_cond_wait()`，`pthread_cond_timedwait()`, `pthread_testcancel()`
- I/O相关：`read()`, `write()`, `open()`, `close()`, `printf()`, `scanf()`
- 等待相关：`wait()`, `waitpid()`, `sleep()`, `usleep()`, `nanosleep()`
- 信号相关：`sigwait()`, `pause()`
- IPC相关：`msgsnd()`, `msgrcv()`

2.手动创建取消点：

```c
void pthread_testcancel(void);
```

- 如果当前有挂起的取消请求，此函数就是一个取消点，线程会终止
- 如果没有挂起请求，此函数无效

## 线程取消时的清理机制：清理函数栈

线程被取消时，可能需要执行清理操作：

- 解锁持有的互斥量
- 释放动态分配的内存
- 关闭打开的文件描述符
- 恢复共享变量状态

1.清理函数栈的概念：

- 线程可以维护一个栈的清理函数
- 当线程被取消时，这些清理函数会自动按注册的逆序执行
- 清理函数执行完毕后，线程才真正终止

2.主要API

```c
pthread_cleanup_push() //将清理函数压入栈顶
pthread_cleanup_pop() //弹出并执行栈顶的清理函数（或仅弹出不执行）
```

## 线程取消完整生命周期

```bash
发送取消请求
    ↓
[pthread_cancel() 返回]
    ↓
取消状态检查
    ↓
├─ 如果禁用取消 → 请求挂起，直到启用
└─ 如果启用取消 → 根据类型决定何时响应
    ├─ 异步取消 → 随时可能响应（危险）
    └─ 延迟取消 → 到达下一个取消点时响应
        ↓
到达取消点
    ↓
执行清理函数栈（逆序）
    ↓
线程终止
```
