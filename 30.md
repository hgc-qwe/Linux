# 总结

## 线程同步

1.原因：线程共享内存空间，多个线程同时读写同一变量时，操作可能被中断，导致数据不一致
2.临界区：访问共享资源的代码片段，一次只能允许一个线程执行
3.竞争条件：多个线程并发放问共享数据，最终结果依赖于线程执行的精确时序

## 互斥量

互斥量用于保护共享变量，确保同一时刻只有一个线程能访问被保护的资源
1.基本概念：

- 互斥量本质是一个锁：线程在进入临界区前锁定互斥量，退出临界区时解锁
- 如果互斥量已被其他进程锁定，试图锁定他的线程会阻塞，直到互斥量被解锁

2.主要API

- `pthread_mutex_init()`初始化互斥量
- `pthread_mutex_destroy()`销毁互斥量
- `pthread_mutex_lock()`锁定互斥量（若已被锁定则阻塞）
- `pthread_mutex_trylock()`尝试锁定（若已被锁定则立即返回EBUSY）
- `pthread_mutex_unlock()`解锁互斥量

3.静态与动态初始化

```c
// 静态初始化
pthread_mutex_t mutex = PTHREAD_MUTEX_INITIALIZER;

// 动态初始化
pthread_mutex_t mutex;
pthread_mutex_init(&mutex, NULL);
```

4.死锁

- 线程A锁定互斥量a，等待互斥量b；线程B锁定互斥量b，等待互斥量a。两者互相等待，永远无法继续
- 统一线程两次锁定同一个互斥量
- 避免方法：所有线程按相同顺序锁定多个互斥量；使用`pthread_mutex_trylock()`

## 条件变量

条件变量用于线程间的通知机制：允许一个线程等待某个条件成立，另一个线程在条件变化时通知等待线程
1.主要API

- `pthread_cond_init()`初始化条件变量
- `pthread_cond_destroy()`销毁条件变量
- `pthread_cond_wait()`等待条件变量（原子性地解锁互斥量并挂起线程）
- `pthread_cond_signal()`唤醒一个等待该条件变量的线程
- `pthread_cond_broadcast()`唤醒所有等待该条件变量的线程
- `pthread_cond_timedwait()`带超时机制的等待

2.使用模式

- 等待线程（消费者）

```c
pthread_mutex_lock(&mutex);
while (condition_is_false) {           
    pthread_cond_wait(&cond, &mutex);  // 原子操作：解锁mutex + 挂起等待
}
// 条件满足，访问共享变量...
pthread_mutex_unlock(&mutex);
```

- 通知线程（生产者）

```c
pthread_mutex_lock(&mutex);
// 修改共享变量，使条件成立...

pthread_cond_signal(&cond);  // 或 pthread_cond_broadcast(&cond);
pthread_mutex_unlock(&mutex);
```
